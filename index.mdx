Agentopia is an autonomous multi-bot platform that orchestrates AI agents for software delivery. Specialized bots — solution architects, developers, code reviewers, QA analysts — collaborate through the A2A protocol with governed execution boundaries and durable workflow orchestration.

## What's shipped

<CardGroup cols={2}>
  <Card title="Multi-Bot Orchestration" icon="users-gear">
    Role-based authority contracts, A2A coordination, per-agent semantic memory, file-based workspace isolation
  </Card>
  <Card title="Workflow Orchestration" icon="diagram-project">
    Temporal-powered durable workflows: plan, dispatch, develop, review, merge. Rework loops, cancellation with structured cleanup
  </Card>
  <Card title="Governed PR Review" icon="shield-check">
    Policy-driven bot review with risk triage, 29 governance tools, finding closure tracking, incremental re-review
  </Card>
  <Card title="Web Application" icon="browser">
    Dual-lane operator UX: Communication + Workflow. Bot management, workflow monitoring, provider/model selection
  </Card>
  <Card title="Provider/Model Control" icon="microchip">
    Multi-provider catalog (Anthropic, OpenAI, Fireworks, OpenRouter). Per-bot provider/model editable after deploy via UI
  </Card>
  <Card title="Infrastructure" icon="server">
    GitOps via ArgoCD, per-environment CI/CD, Prometheus observability, custom Rust LLM proxy
  </Card>
</CardGroup>

## Platform components

| Component | Purpose |
|---|---|
| **bot-config-api** | Control plane — bot provisioning, deployment, workflow coordination, governance |
| **agentopia-gateway** | Bot runtime — LLM orchestration, tool execution, channel integration |
| **agentopia-llm-proxy** | LLM routing — multi-provider abstraction, failover, token counting (Rust) |
| **mem0-api** | Memory service — semantic search (Qdrant), entity graph (Neo4j) |
| **agentopia-ui** | Web interface — bot management, workflow monitoring, knowledge management |
